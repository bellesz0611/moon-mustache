FAQ

Project boundaries and adoption questions, answered directly.

This page condenses the most common questions a reviewer, adopter, or teammate might ask about Moon Mustache.

Why Mustache?

Because it is small, widely understood, easier to validate against upstream expectations, and easier to trust than a custom logic-heavy template syntax.

Why not make a bigger template language?

The project intentionally trades breadth for predictability. It is optimized for scaffolding, config rendering, content generation, and reusable MoonBit tooling workflows.

How much real adoption evidence is there?

The repository now includes four consumer-style demos, a package publication, a playground, a static site, release-history notes, and dedicated consumer-proof workflow automation.

What is still not fully proven?

Independent external adopters, long-duration maintenance history, and broader community signals are still growing. The repository documents those limits explicitly instead of hiding them.
