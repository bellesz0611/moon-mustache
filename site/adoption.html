Adoption Surfaces

Multiple consumer scenarios make the reuse story easier to trust.

Moon Mustache is no longer represented by a single example package. The repository now includes several consumer-style proof surfaces for different workflow families.

Release communications

`downstream_consumer/` shows profile-aware bundle generation for release notes and announcement flows.

Ops rollout generation

`adoption_demo/` demonstrates environment-style multi-file output for deployment and ownership artifacts.

Content pipeline

`content_pipeline_demo/` generates Markdown, text, and HTML outputs from the same release-oriented input data.

Starter repository scaffolding

`starter_repo_demo/` demonstrates generated starter files such as README, moon.mod, .gitignore, and starter source code.

Companion-repo blueprint

`companion_repo_blueprint/` shows what a split-out independent repository consumer could look like.

Workflow-backed evidence

The main CI, release-readiness, and dedicated consumer-proofs workflow all exercise adoption-oriented surfaces.
