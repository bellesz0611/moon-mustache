# Support

## Where to ask for help

Use the repository issue tracker for:

- reproducible bugs
- feature requests
- documentation gaps
- compatibility questions tied to specific templates or outputs

Use private reporting paths for security-sensitive issues. See `SECURITY.md`.

## What to include

To get faster help, include:

- MoonBit version
- target (`wasm-gc` or `js`)
- minimal template
- context JSON or MoonBit-side input
- partials if used
- expected output vs actual output
- relevant command or API entrypoint

## Best-effort support areas

This project is best positioned to help with:

- Mustache rendering behavior
- diagnostics and missing-variable handling
- CLI and bundle workflows
- compatibility and spec coverage
- playground demo issues

## Out-of-scope support

The repository is less suitable for:

- unrelated general-purpose template design advice
- non-Mustache language extensions
- production hosting issues unrelated to the repository's own code
